# v0.9.7 受入テスト方針

ManageAdTool v0.9.7 は、実運用担当者による **受入テスト版** として扱います。

v0.9.6 は、開発側打鍵テストで見つかった指摘の修正結果を取り込み済みの版です。v0.9.7 では、v0.9.6 の修正結果を前提に、実運用担当者が実際の手順・画面表示・説明文・運用上の分かりやすさを確認します。

---

## 基本方針

- 参照専用 AD 確認支援ツールである方針を維持します。
- 受入テスト中に、AD 更新機能・グループ更新機能・GPO 操作などの高リスク機能は追加しません。
- v0.9.7 で取り込む修正は、v1.0.0 前の受入判断に必要な軽微な修正・説明改善を中心にします。
- 機能追加に近い指摘、大きな設計変更を伴う指摘、運用リスクが高い指摘は v1.0.0 以降に回します。

---

## v0.9.7 系で取り込む指摘の範囲

同じ v0.9.7 系で取り込む対象は、原則として次の範囲です。

- 文言修正
- 表示調整
- 手順書修正
- 参照専用であることの説明改善
- 軽微な不具合修正

これらは、受入テストでの誤解防止、操作手順の明確化、表示の見やすさ改善、v1.0.0 前に解消すべき軽微な不具合の修正を目的とします。

---

## v1.0.0 以降に回す指摘の範囲

次の指摘は、v0.9.7 の受入テスト中には原則として取り込まず、v1.0.0 以降の検討事項として扱います。

- 新機能追加
- CSV 出力拡張
- ネストグループ展開
- 大きな UI 変更
- AD 更新機能
- グループ更新機能
- GPO 操作

これらは、要件整理・設計・権限設計・検証範囲の再定義が必要になるため、受入テスト版の安定化対象から外します。

---

## 受入テストでの指摘分類

受入テストで出た指摘は、次の A / B / C に分類します。

| 区分 | 扱い | 例 |
|---|---|---|
| A | v1.0.0 前に必ず対応 | 受入判断に必要な文言修正、参照専用であることの誤解を招く表示、軽微だが運用手順を阻害する不具合 |
| B | v0.9.7 内で可能なら対応 | 表示調整、手順書の補足、優先度が中程度の説明改善 |
| C | v1.0.0 以降 | 新機能追加、CSV 出力拡張、ネストグループ展開、大きな UI 変更、AD 更新機能、グループ更新機能、GPO 操作 |

分類に迷う場合は、参照専用 AD 確認支援ツールとして v1.0.0 前に必要かどうか、また v0.9.7 の安定性を損なわないかを基準に判断します。

---

## 記録時の推奨項目

受入テストの指摘を記録する場合は、次の項目を残します。

| 項目 | 内容 |
|---|---|
| 指摘 ID | 一意に追跡できる番号 |
| 区分 | A / B / C |
| 対象画面・資料 | 指摘が発生した画面、操作、ドキュメント |
| 指摘内容 | 実運用担当者からの指摘 |
| 対応方針 | v0.9.7 で対応 / v0.9.7 内で可能なら対応 / v1.0.0 以降 |
| 対応結果 | 修正内容、または先送り理由 |

---

## v0.9.6 との関係

- v0.9.6: 開発側打鍵テストの修正結果を取り込み済みの版。
- v0.9.7: v0.9.6 の結果を前提に、実運用担当者が受入テストを行う版。
- v1.0.0 以降: 受入テストで見つかった機能追加・大きな設計変更・高リスク操作を検討する版。
