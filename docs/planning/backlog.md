## v0.9.x 完了条件（v1.0.0 着手前に確認）

以下をすべて実AD環境で確認してから次のフェーズに進んでください（`docs/operation/validation-log-viewer.md` 参照）。

- [ ] ログ確認タブが表示される
- [ ] 参照ログ（audit.jsonl）を選択して読み込み・表示できる
- [ ] 認証ログ（auth.jsonl）を選択して読み込み・表示できる
- [ ] 書き込みログ（write-audit.jsonl）を選択して読み込み・表示できる
- [ ] ログファイルが存在しない場合にエラーで落ちない（エラーメッセージ表示）
- [ ] 大きなログファイル（1000行超）で末尾からの取得・上限警告が正常に動作する
- [ ] 壊れたJSON行が含まれていても他の行が正常に表示される（解析エラー耐性）
- [ ] 成功/失敗フィルタが正常に動作する
- [ ] 操作種別・対象名キーワードフィルタが正常に動作する
- [ ] 1件選択して整形済みJSONが表示される
- [ ] passwordキーが存在する行でマスク（`***`）されることを確認する
- [ ] CSV出力が文字化けしない（BOM付きUTF-8）
- [ ] ログフォルダを開けるボタンが動作する
- [ ] ログ確認タブの操作で write-audit.jsonl に新規エントリが記録されないことを確認する（AD書き込みなし）
- [ ] 参照ログ（audit.jsonl）に `executor` / `machineName` フィールドが含まれることを確認する

## v1.0.0 候補（優先度順・未確定）

v1.0.0 は機能追加より**既存機能の安定化・実運用準備**を主目的とします。v1.0.0 前に新しい AD 更新対象属性を増やさない方針を優先して検討しています。

### 実AD検証結果の反映
- v0.8.x / v0.9.x の実AD検証で判明した問題の修正
- 各検証手順（validation-*.md）の未確認項目の消化

### UI整理
- ボタン配置・ラベルの見直し
- 長いディスエーブル理由の表示改善

### リリースパッケージ整理
- self-contained ZIP の配布手順確認
- `config-samples/` の最終確認・整理

### 操作説明書整備
- 利用者向けの簡易操作ガイドの追加・整理

### 既存機能の安定化
- キャッシュ評価（DirectoryReadOnly 実運用後に評価）
  - ユーザー選択切替ごとの `memberOf` 再取得削減
  - セッション内メモリキャッシュ（TTL: 30〜120秒）
- `accountExpires` 更新設計継続（`docs/design/design-account-expiration.md` 参照）

### 編集機能の拡張（v1.0.0 後に検討）
- ユーザー更新対象属性の追加（要件定義次第）
- オペレーション支援の拡張（`OperationChecklistItems` の UI 反映）
