# グループ参照・メンバー編集 検証手順書 (v0.6.0)

## 概要

本手順書は ManageAdTool v0.6.0 のグループ参照機能・グループメンバー追加削除機能を実 AD 環境で検証するための手順を記述します。

**参照対象**: グループの Name / DistinguishedName / Description / ユーザーメンバー / コンピュータメンバー / ネストグループ / memberOf  
**更新対象**: グループの `member` 属性（ユーザーの追加・削除のみ）  
**禁止操作**: グループ作成・削除・リネーム / description 編集 / グループをグループに追加 / コンピュータをグループに追加 / GPO編集 / OU移動 / 一括更新  
**前提条件**: `DirectoryReadOnly` モード + Domain Admins セッション + `EditableGroupOuDns` 設定

---

## 事前準備

### appsettings.json の設定

`appsettings.GroupMembershipEdit.sample.json` を参考に設定する。

```json
{
  "AppPolicy": {
    "ServiceMode": "DirectoryReadOnly",
    "EditorAuthMode": "DomainAdmins",
    "AdminGroupDn": "CN=Domain Admins,CN=Users,DC=example,DC=local",
    "EditableGroupOuDns": [
      "OU=Groups,DC=example,DC=local"
    ],
    "ProtectedGroupNames": ["Domain Admins", "Enterprise Admins", "Schema Admins", "Administrators"],
    "ProtectedGroupDns": [
      "CN=Domain Admins,CN=Users,DC=example,DC=local",
      "CN=Enterprise Admins,CN=Users,DC=example,DC=local"
    ]
  }
}
```

> **EditableGroupOuDns の設定について**: `EditableGroupOuDns` が空の場合はグループメンバーの更新を実行できません（セーフガード）。編集対象グループを特定の OU に限定することで、意図しないグループへの変更を防止します。

> **ProtectedGroupNames / ProtectedGroupDns の設定について**: Domain Admins・Enterprise Admins・Schema Admins・Administrators など特権グループは必ずここに登録してください。登録されたグループはメンバーの追加・削除が不可になります。名前ベースと DN ベースを両方登録することを推奨します。

### テスト環境

AD 上に以下を用意する:

- `GG_TestGroup`（検証用グループ。`EditableGroupOuDns` 内の OU に配置）
- `GG_ProtectedTest`（`ProtectedGroupNames` または `ProtectedGroupDns` に登録した保護グループ）
- テスト用ユーザー `test.user01`（GG_TestGroup の既存メンバー）
- テスト用ユーザー `test.user02`（GG_TestGroup の非メンバー）

## 検証項目

### 1. グループ検索

| 項目 | 手順 | 期待結果 |
|------|------|---------|
| 基本検索 | 「グループ参照・メンバー編集」タブを開き、グループ名の一部（2文字以上）を入力して「検索」 | 該当グループが一覧に表示される（Name / DistinguishedName 列） |
| 1文字検索禁止 | 1文字を入力して「検索」 | 「2文字以上入力してください」メッセージ |
| 上限到達 | 広範な検索語で MaxSearchResults 件数に達する場合 | 上限件数と超過メッセージを表示 |

### 2. グループ詳細表示

グループ検索結果から行を選択すると、左ペインに詳細が表示される。

| 表示エリア | 表示内容 |
|-----------|---------|
| グループ情報（上部テキスト） | Name / DistinguishedName / Description（空の場合は非表示） |
| ユーザーメンバー（DataGrid） | SamAccountName / DisplayName / Mail / Department / Enabled（ユーザー種別のみ表示） |
| その他メンバー・所属（下部テキスト） | コンピュータメンバー数と名前一覧 / ネストグループ数と名前一覧 / memberOf（所属グループ名一覧） |

**確認項目:**

| 項目 | 手順 | 期待結果 |
|------|------|---------|
| グループ情報表示 | 検索結果からグループを選択 | Name / DN が表示される。Description がある場合は表示、ない場合は非表示 |
| ユーザーメンバー表示 | 同上 | ユーザーメンバーの DataGrid に SamAccountName / DisplayName / Mail / Department / Enabled が表示される |
| コンピュータメンバー表示 | コンピュータが member に含まれるグループを選択 | 「コンピュータメンバー (N台): 名前一覧」が下部テキストに表示される |
| ネストグループ表示 | グループが member に含まれるグループを選択 | 「ネストグループ (N件): 名前一覧」が下部テキストに表示される |
| memberOf 表示 | 他グループに所属しているグループを選択 | 「所属グループ/memberOf (N件): 名前一覧」が下部テキストに表示される |

### 3. 編集ブロック確認

右ペインの「メンバー編集」エリアで入力がブロックされることを確認する。

| 項目 | 手順 | 期待結果 |
|------|------|---------|
| グループ未選択 | グループを選択しない状態 | 追加用テキストボックス・ボタンが非活性。理由「グループ未選択」が表示される |

## 更新フロー（正常系）

```
1. 変更予定の作成
   - ユーザー追加: SAM を入力 → 「検索して追加予定へ」 → AD で存在確認 → 追加予定リストへ
   - ユーザー削除: DataGrid でユーザー選択 → 「選択を削除予定へ」 → 削除予定リストへ

2. 差分確認
   - 「差分確認」ボタンをクリック → OutputBox に変更予定一覧を表示
   - 「限定更新実行」ボタンが有効化される

3. 再認証（LDAP バインド）
   - 「限定更新実行」クリック → ReAuthDialog（ユーザー名・パスワード入力）
   - Domain Admins グループメンバーであることを確認
   - セッション開始ユーザーと一致することを確認

4. 実行前確認
   - ConfirmGroupMemberUpdateDialog を表示
     - 対象グループ名 / DN / 実行端末 / 編集セッションユーザー
     - 追加するユーザー（緑）/ 削除するユーザー（赤）
   - 「この内容でADを更新する」で次へ進む

5. 更新前 AD 再取得・整合性チェック
   - AD からグループの現在メンバー一覧を再取得
   - 追加予定ユーザーが既にメンバーでないか確認（不一致なら中止）
   - 削除予定ユーザーが依然メンバーであるか確認（不一致なら中止）

6. 更新
   - LDAP で `member` 属性を操作（追加: Add / 削除: Remove）
   - 再認証パスワードは書き込みに即時使用・破棄（保存しない）

7. 更新後 AD 再取得
   - グループの最新状態を AD から再取得
   - ユーザーメンバー DataGrid を更新

8. 監査ログ出力
   - write-audit.jsonl に追記（targetType: Group / operationName: UpdateGroupMembers）
```

- write-audit.jsonl の `operationName: "UpdateGroupMembers"` エントリを参照して変更内容を確認する
