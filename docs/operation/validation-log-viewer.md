# ログ確認機能 検証手順

v0.9.0 で追加した「ログ確認」タブの動作を実AD環境で確認する手順です。

## 前提条件

- `ServiceMode: "DirectoryReadOnly"` または `"InMemory"` で起動済み
- `LogPath` が書き込み可能なパスに設定済み
- 事前に参照操作・認証操作・書き込み操作を実行し、各ログファイルにエントリが存在する状態にしておく

---

## 検証項目

### 1. タブ表示

- [ ] 「ログ確認」タブが画面上に表示される

### 2. ログ種別選択と読み込み

- [ ] 「参照ログ (audit.jsonl)」を選択して「読み込み」を押すと、audit.jsonl の内容が DataGrid に表示される
- [ ] 「認証ログ (auth.jsonl)」を選択して「読み込み」を押すと、auth.jsonl の内容が DataGrid に表示される
- [ ] 「書き込みログ (write-audit.jsonl)」を選択して「読み込み」を押すと、write-audit.jsonl の内容が DataGrid に表示される
- [ ] ログファイルが存在しない場合、「ファイルが存在しません」旨のエラーが表示される
- [ ] 読み込み後、件数が状態バーに表示される

### 3. DataGrid 表示内容

- [ ] 日時 / 操作 / 対象 / 実行者・編集者 / 成否 / 備考 の6列が表示される
- [ ] 「成否」列: 成功操作が「○」、失敗操作が「✕」、成否なし（参照操作等）が「-」と表示される
- [ ] 「実行者・編集者」列: `editorUser` があればそちらを優先し、なければ `executor` が表示される

### 4. フィルタリング

- [ ] 開始日・終了日を指定して「フィルター適用」を押すと、指定範囲のエントリのみ表示される
- [ ] 「成否: 成功」を選択するとSuccess=trueのエントリのみ、「失敗」を選択するとSuccess=falseのみ表示される
- [ ] 操作キーワードに文字列を入力すると、「操作」列に一致するエントリのみ表示される
- [ ] 対象キーワードに文字列を入力すると、「対象」列に一致するエントリのみ表示される
- [ ] 「クリア」を押すとフィルターがリセットされ、全件が表示される

### 5. 詳細JSON表示

- [ ] DataGrid で行を選択すると、下部の詳細ペインに整形済みJSONが表示される
- [ ] 改行・インデント付きで読みやすく整形されている
- [ ] `"password"` キーの値が `"***"` にマスクされている（auth.jsonl の認証エントリ等で確認）

### 6. クリップボードコピー

- [ ] 詳細ペインの「コピー」ボタンを押すと、JSONがクリップボードにコピーされる
- [ ] テキストエディタに貼り付けて内容を確認できる

### 7. CSV出力

- [ ] 「CSV出力」ボタンを押すと保存ダイアログが表示される
- [ ] CSVを保存し、Excel等で開いて内容を確認する（ヘッダー7列: 日時/操作/対象/実行者・編集者/成否/備考/種別）
- [ ] BOM付きUTF-8で出力されており、日本語が文字化けしない
- [ ] フィルター適用中はフィルター結果がCSV出力される
- [ ] 読み込み前にCSV出力ボタンを押した場合、エラーメッセージが表示される

### 8. フォルダを開く

- [ ] 「フォルダを開く」ボタンを押すと、ログ格納フォルダがエクスプローラーで開く

### 9. 最大表示件数

- [ ] ログファイルの行数が `MaxLogDisplayRows`（デフォルト 1000）を超える場合、末尾から 1000 件のみ表示される
- [ ] 超過時に「最新 X 件のみ表示しています」旨の警告が表示される

### 10. 解析エラー耐性

- [ ] ログファイルに不正な JSON 行が含まれていても、他の行は正常に表示される
- [ ] 解析エラー行は「操作」列に "(解析エラー)" と表示される

### 11. AD更新が実行されないこと（最重要）

- [ ] ログ確認タブのいかなる操作も、ADへの書き込みを行わない
- [ ] このタブの操作後に `write-audit.jsonl` を確認し、新たな更新ログが記録されていないことを確認する

### 12. 参照ログへの executor / machineName 記録

- [ ] 参照操作（ユーザー検索など）後に audit.jsonl を確認し、`"executor"` と `"machineName"` フィールドが含まれる
- [ ] `executor` の値がアプリ起動ユーザー（domain\user 形式）と一致する
- [ ] `machineName` の値が実行端末のホスト名と一致する

### 13. InMemory モードでの動作確認

- [ ] `ServiceMode: "InMemory"` でもログ確認タブが表示される
- [ ] InMemory モードでもログファイルが存在すれば読み込み・表示できる
