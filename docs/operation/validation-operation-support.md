# オペレーション支援機能 検証手順

v0.7.0 で追加した「オペレーション支援」タブの動作を実AD環境で確認する手順です。

---

## 前提条件

- 起動時ダイアログで `DirectoryReadOnly` または `InMemory` を選択済み
- `EnableOperationSupport: true`（デフォルト有効）
- `LogPath` が書き込み可能なパスに設定済み

---

## 検証項目

### 1. タブ表示

- [ ] 「オペレーション支援」タブが画面上に表示される
- [ ] `EnableOperationSupport: false` を設定した場合、タブが非表示になる（または無効化される）

### 2. ユーザー検索・選択

- [ ] 検索ボックスにキーワードを入力し「検索」を押すと、一覧にユーザーが表示される
- [ ] 一覧からユーザーを選択すると、現在値欄（sAMAccountName / 表示名 / 姓 / 名 / メール）に値が表示される
- [ ] 現在値欄が読み取り専用（編集不可）であることを確認する
- [ ] 所属グループ欄に現在の所属グループ名が一覧表示される

### 3. 変更後の値の入力

- [ ] 「変更後」欄（表示名 / 姓 / 名 / メール）に任意の値を入力できる
- [ ] 現在値と同じ値を入力してもエラーにならない

### 4. グループ追加予定

- [ ] 「追加予定グループ名」に存在するグループ名を入力し「追加予定へ」を押すと、追加予定リストに追加される
- [ ] 同じグループを二度追加しようとしても重複追加されない
- [ ] 削除予定リストに既に入っているグループは追加予定に追加されない（矛盾防止）
- [ ] 追加予定リストで項目を選択し「選択を追加予定から削除」を押すと、リストから除去される

### 5. グループ削除予定

- [ ] 「削除予定グループ名」に入力し「削除予定へ」を押すと、削除予定リストに追加される
- [ ] 同じグループを二度追加しようとしても重複追加されない
- [ ] 追加予定リストに既に入っているグループは削除予定に追加されない（矛盾防止）
- [ ] 削除予定リストで項目を選択し「選択を削除予定から削除」を押すと、リストから除去される

### 6. 確認チェックリスト

- [ ] チェックリスト（対象ユーザー確認・属性確認・グループ確認・関係部署確認等）が表示される
- [ ] 各チェックボックスをオン・オフできる
- [ ] チェック状態は画面上のみ保持され、ユーザー選択を切り替えるとリセットされる

### 7. 変更予定サマリー生成

- [ ] 「変更予定サマリー生成」ボタンを押すと、サマリーテキストが生成されてテキストボックスに表示される
- [ ] サマリーに対象ユーザーの sAMAccountName・現在の表示名が含まれる
- [ ] 変更後の値が現在値と異なる場合のみ「属性変更予定」としてサマリーに記載される
- [ ] 変更後の値が現在値と同じ場合は「属性変更予定」に記載されない
- [ ] グループ追加予定・削除予定がサマリーに記載される
- [ ] サマリーに「実際の更新は各タブから実行してください」旨の注記が含まれる

### 8. クリップボードコピー

- [ ] 「クリップボードにコピー」ボタンを押すと、サマリーテキストがクリップボードにコピーされる
- [ ] テキストエディタ等に貼り付けて内容が正しいことを確認する

### 9. AD更新が実行されないこと（最重要）

- [ ] オペレーション支援タブのいかなる操作も、AD への書き込みを行わない
- [ ] サマリー生成・コピー後に `write-audit.jsonl` を確認し、このタブの操作に起因する更新ログが記録されていないことを確認する
- [ ] 実際のユーザー属性がADで変わっていないことを確認する
- [ ] 実際のグループメンバーシップがADで変わっていないことを確認する

### 10. 参照ログへの記録

- [ ] サマリー生成後に `audit.jsonl` を確認し、`"action": "OperationPlanCreated"` のエントリが記録されている
- [ ] ログに `executor`（操作者）・`targetUser`・`plannedAttributeChanges`・`plannedGroupAdds`・`plannedGroupRemoves` が含まれる

### 11. ユーザー切り替え時のリセット

- [ ] 別のユーザーを選択すると、変更後入力欄・グループ追加削除予定・サマリーテキストがクリアされる

### 12. 対象ユーザー未選択時の動作

- [ ] ユーザーを選択せずに「変更予定サマリー生成」を押すと、エラーまたは「未選択」を示すメッセージが表示される

### 13. InMemory モードでの動作確認

- [ ] 起動時ダイアログで `InMemory` を選択 でも上記の操作が正常に動作する（AD接続なしで検証可能）
- [ ] InMemory モードでサマリー生成した場合も `audit.jsonl` にログが記録される
