# ManageAdTool トラブルシューティング（v0.9.4）

この文書は、ManageAdTool 利用時によくある問題と確認観点をまとめたものです。実 AD 更新が関係する問題では、原因が不明なまま再実行を繰り返さず、設定・対象 OU・ログを確認してから対応してください。

---

## 1. アプリが起動しない

確認点:

- 配布 ZIP を展開せずに直接実行していないか。
- `ManageAdTool.exe` と必要ファイルが同じフォルダにあるか。
- `appsettings.json` の JSON 構文が壊れていないか。
- 実行端末のセキュリティ製品や AppLocker 等でブロックされていないか。
- ログ出力先フォルダにアクセスできるか。

対応:

1. ZIP をローカルフォルダへ展開します。
2. `appsettings.json` をサンプルから再作成して起動確認します。
3. `InMemory` 設定で起動するか確認します。
4. 端末側の実行制限を管理者に確認します。

---

## 2. AD検索できない

確認点:

- 起動時ダイアログで `DirectoryReadOnly` を選択しているか。
- 端末がドメインネットワークに接続されているか。
- DNS が AD ドメインを解決できるか。
- 実行ユーザーに参照権限があるか。
- AD 側またはネットワーク側で一時障害がないか。

`InMemory` の場合は実 AD に接続しないため、実 AD データは検索できません。

---

## 3. ユーザーが検索結果に出ない

確認点:

- キーワードが正しいか。
- 検索条件を絞りすぎていないか。
- 対象ユーザーが無効化されており、無効ユーザーを含める条件が必要ではないか。
- `MaxSearchResults` 上限により結果が省略されていないか。
- 対象ユーザーが想定 OU に存在するか。

対応:

- sAMAccountName など一意に近い値で検索します。
- 条件を減らして再検索します。
- AD 管理ツール等で対象 DN を確認します。

---

## 4. コンピュータが検索結果に出ない

確認点:

- コンピュータ名、DNS ホスト名、sAMAccountName の入力が正しいか。
- OS 条件や description 有無条件を絞りすぎていないか。
- 無効コンピュータを除外していないか。
- 対象が `ExcludedComputerNames` に該当していないか（編集不可理由にも影響します）。

---

## 5. グループが検索結果に出ない

確認点:

- グループ名または DN の入力が正しいか。
- 検索キーワードが短すぎないか。
- グループが想定 OU に存在するか。
- 実行ユーザーに参照権限があるか。

---

## 6. Domain Admins認証に失敗する

確認点:

- `EditorAuthMode` が `DomainAdmins` になっているか。
- `AdminGroupDn` が実 AD の Domain Admins DN と一致しているか。
- 入力したアカウントが Domain Admins のメンバーか。
- ネストグループ判定が運用要件と一致しているか。
- パスワード、ロックアウト、期限切れ、ドメイン指定に問題がないか。
- `auth.jsonl` に失敗ログが出ているか。

パスワードはログに出力されません。

---

## 7. 編集ボタンが有効にならない

よくある原因:

- 対象を選択していない。
- `AllowedTargetOuDns` / `AllowedComputerOuDns` / `EditableGroupOuDns` が未設定。
- 対象 DN が許可 OU 配下ではない。
- 対象が `ExcludedSamAccountNames` / `ExcludedComputerNames` に該当する。
- 対象グループが `ProtectedGroupNames` / `ProtectedGroupDns` に該当する。
- `EditableAttributes` または `EditableComputerAttributes` が不足している。
- `DirectoryReadOnly` で Domain Admins セッションが有効ではない。

対応:

1. 対象選択と DN を確認します。
2. appsettings.json の OU / 除外 / 保護設定を確認します。
3. 必要に応じて再認証します。
4. 設定変更後はアプリを再起動します。

---

## 8. 更新できない

確認点:

- Domain Admins セッションが期限切れではないか。
- 更新対象が許可 OU 配下か。
- ユーザー属性は mail / displayName / sn / givenName のみか。
- コンピュータ更新は description のみか。
- グループ更新はユーザーの追加・削除のみか。
- 保護グループや除外対象に該当していないか。
- AD 側の属性制約や権限で拒否されていないか。
- `write-audit.jsonl` に失敗ログが出ているか。

対応:

- エラー内容と監査ログを確認します。
- 同じ操作を繰り返す前に、設定と対象 DN を確認します。
- 不明な場合は更新を中断し、管理者が AD 側の状態を確認してください。

---

## 9. AllowedTargetOuDns / AllowedComputerOuDns / EditableGroupOuDns の設定ミス

症状:

- 編集ボタンが有効にならない。
- 許可 OU 外と表示される。
- 想定外の対象が編集可能に見える。

確認点:

- DN のスペル、カンマ、OU 階層、DC 部分が正しいか。
- ユーザー用、コンピュータ用、グループ用の OU を混同していないか。
- 本番 OU を広く指定しすぎていないか。

運用方針:

- まず検証用 OU のみを指定してください。
- 設定変更後はアプリを再起動してください。

---

## 10. ProtectedGroupNames により編集できない

症状:

- グループメンバー編集ができない。
- 保護グループとして扱われる。

確認点:

- 対象グループ名が `ProtectedGroupNames` に一致していないか。
- 対象グループ DN が `ProtectedGroupDns` に一致していないか。
- 同名グループが存在しないか。

保護設定は安全側の制御です。高権限グループを編集対象にするために安易に保護設定を外さないでください。

---

## 11. ログが出力されない

確認点:

- `LogPath` のフォルダに書き込み権限があるか。
- フォルダが存在しない場合に作成できる権限があるか。
- セキュリティ製品により書き込みがブロックされていないか。
- ディスク空き容量が不足していないか。

ログ書き込み失敗は、参照操作や認証操作を妨げない設計の箇所があります。監査要件がある場合は、ログ出力先の権限を事前に検証してください。

---

## 12. ログ確認タブにログが表示されない

確認点:

- 正しいログ種別を選択しているか。
- `audit.jsonl` / `auth.jsonl` / `write-audit.jsonl` が存在するか。
- `LogPath` と実際のログ保存先が一致しているか。
- 日付範囲、成否、操作種別、対象名のフィルタで除外されていないか。
- `MaxLogDisplayRows` が小さすぎないか。
- 壊れた JSON 行がある場合でも、解析エラー行として表示されるか。

---

## 13. CSVが文字化けする

確認点:

- Excel で直接開いて文字コードが誤判定されていないか。
- 「データの取得」等で UTF-8 として取り込んでいるか。
- エディタ側の文字コード設定が UTF-8 になっているか。

CSV には内部情報が含まれる場合があります。文字化け確認のために外部サービスへアップロードしないでください。

---

## 14. GPOシミュレーション結果が想定と違う

確認点:

- 入力したユーザー / コンピュータが正しいか。
- 対象の OU が想定どおりか。
- GPO リンクの有効 / 無効、強制適用の状態を確認したか。
- セキュリティフィルタ、WMI フィルタ、継承ブロック、ループバック処理、サイトリンクの影響を期待していないか。

---

## 15. GPOシミュレーションはRSOP代替ではない

ManageAdTool の GPOシミュレーションは、OU リンクから推定される GPO 一覧を確認する簡易機能です。

代替しないもの:

- GPMC の RSOP（ポリシーの結果セット）
- `gpresult /R`
- セキュリティフィルタ評価
- WMI フィルタ評価
- 継承ブロック評価
- ループバック処理評価
- サイトリンク評価

厳密な適用結果確認が必要な場合は、GPMC や `gpresult` を使用してください。

---

## 16. self-contained版なのに起動できない場合の確認点

self-contained 版は .NET Desktop Runtime の別途インストールを前提としません。それでも起動できない場合は次を確認してください。

- OS が対象の Windows 環境か。
- ZIP 展開後に全ファイルが揃っているか。
- 実行ファイルがブロック解除されているか。
- セキュリティ製品や実行制御でブロックされていないか。
- 配置パスに日本語・空白・長すぎるパス等の影響がないか。
- `appsettings.json` が壊れていないか。
- ログ出力先にアクセスできるか。

切り分けとして、別フォルダに展開し、`InMemory` 設定で起動確認してください。
