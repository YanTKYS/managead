# 編集者ログイン・Domain Admins 判定 検証手順（v0.3.0）

本手順は **v0.3.0 の認証機能**（別ユーザーログイン・Domain Admins 判定・編集セッション）を  
実 AD 環境で検証するためのものです。

> **注意**: v0.3.0 では実 AD 更新機能は未実装です。  
> 本手順では AD 更新は行いません。認証・セッション・ログ記録の動作のみを確認します。

---

## 1. 事前条件

- 閉域端末に ManageAdTool v0.3.0 の成果物が配置済みであること
- 実 AD へ参照可能なネットワーク疎通があること
- 検証用 Domain Admins メンバーアカウント（直接所属）が用意されていること
- 検証用 Domain Admins 非メンバーアカウントが用意されていること
- ネスト所属を確認する場合は、中間グループ経由でメンバーになっているアカウントも用意すること

---

## 2. appsettings.json の設定

`EditorAuthMode` / `AdminGroupDn` を設定します。

```json
{
  "AppPolicy": {
    "EditorAuthMode": "DomainAdmins",
    "AdminGroupDn": "CN=Domain Admins,CN=Users,DC=example,DC=local",
    "AllowNestedAdminGroupMembership": false,
    "EditSessionMinutes": 15,
    "AllowedTargetOuDns": [
      "OU=ValidationUsers,OU=ManageAdTool,DC=example,DC=local"
    ],
    "ExcludedSamAccountNames": [
      "administrator",
      "krbtgt"
    ],
    "EditableAttributes": ["mail", "displayName", "sn", "givenName"],
    "LogPath": "C:\\ProgramData\\ManageAdTool\\logs\\audit.jsonl",
    "MaxSearchResults": 200,
    "MaxLogDisplayRows": 1000
  }
}
```

## 4. 判定基準

- Domain Admins 直接所属での認証成功・セッション開始が確認できること
- 誤パスワード・非メンバーでの認証拒否が確認できること
- `auth.jsonl` にパスワードが記録されていないこと
- セッション期限切れでコントロールが無効化されること
- 「更新不可」ボタンが押下可能であっても AD 更新が実行されないこと

---

## 5. 注意事項

- 本手順では実 AD 更新は行いません。v0.3.0 では更新機能は未実装です。
- `auth.jsonl` には認証ユーザー名・成否・タイムスタンプのみ記録されます。パスワードは記録されません。
- セッションのタイムアウト検証のみ `EditSessionMinutes` を短縮して実施してください。通常運用では元の値（15 分程度）に戻してください。
