# ManageAdTool 利用者向け操作説明書（v0.9.4）

対象読者: ManageAdTool を操作する情報システム担当者

> **重要**: 本書は操作手順を整理するものです。ManageAdTool は Active Directory の全管理操作を行うツールではありません。更新できる範囲は、ユーザー属性（mail / displayName / sn / givenName）、コンピュータ description、ユーザーに限定したグループメンバー追加・削除のみです。

---

## 1. 起動方法

1. 管理者から配布されたフォルダを開きます。
2. `ManageAdTool.exe` を実行します。
3. 初回利用時は、管理者から指定された `appsettings.json` が同じフォルダに配置されていることを確認します。
4. 起動時の ServiceMode 選択ダイアログで、管理者から指定されたモードを選択します。

起動しない場合は `troubleshooting.md` の「アプリが起動しない」「self-contained版なのに起動できない場合の確認点」を確認してください。

---

## 2. ServiceMode の選択

| ServiceMode | 用途 | 注意 |
|---|---|---|
| `InMemory` | デモ・画面確認用 | 実 AD 接続・更新は行いません。表示されるデータはダミーです。 |
| `DirectoryReadOnly` | 実 AD 参照 + Domain Admins セッション有効時の限定更新 | 更新は設定で許可された OU / 属性 / グループに限定されます。 |

ServiceMode は `appsettings.json` ではなく起動時に選択します。操作前に、管理者から指定されたモードで起動していることを確認してください。`InMemory` の結果を実 AD の結果として扱わないでください。

---

## 3. ユーザー検索

1. ユーザー検索タブを開きます。
2. sAMAccountName、表示名、氏名、メールアドレスなど、判別しやすいキーワードを入力します。
3. 必要に応じて無効ユーザー表示の条件を指定します。
4. 検索を実行します。
5. 検索結果が多い場合は、キーワードを具体化するか、管理者が設定した `MaxSearchResults` の上限を確認してください。

検索結果に出ない場合は、対象が許可 OU 外、無効ユーザー除外、検索条件過多、AD 接続問題の可能性があります。

---

## 4. ユーザー詳細表示

1. ユーザー検索結果から対象ユーザーを選択します。
2. 詳細欄で sAMAccountName、表示名、DN、メール、姓、名、所属グループなどを確認します。
3. 更新作業前に、対象ユーザーが意図したユーザーであることを DN と所属情報で再確認します。

---

## 5. ユーザー属性限定編集

更新対象は **mail / displayName / sn / givenName のみ** です。sAMAccountName、userPrincipalName、cn、部署、役職、アカウント有効期限、無効化、パスワードリセット等は更新できません。

### 5.1 編集前確認

1. 対象ユーザーを検索・選択します。
2. 対象ユーザーが `AllowedTargetOuDns` 配下であることを確認します。
3. 除外アカウント（`ExcludedSamAccountNames`）に該当しないことを確認します。
4. 変更依頼内容と現状値を照合します。

### 5.2 差分確認

1. mail / displayName / sn / givenName の新しい値を入力します。
2. 差分確認を行い、変更前・変更後の内容を確認します。
3. 差分がない場合は更新しないでください。
4. 対象外属性を更新する目的で備考欄や別属性を代用しないでください。

### 5.3 再認証

1. 更新には Domain Admins セッションが必要です。
2. 再認証ダイアログで管理者アカウントを入力します。
3. パスワードは認証・更新に即時利用され、設定ファイルやログには保存されません。
4. セッションには有効期限があります。期限切れの場合は再認証してください。

### 5.4 更新実行と更新後確認

1. 確認画面で対象ユーザー、対象 DN、変更属性、変更前後の値を再確認します。
2. 更新を実行します。
3. 更新後、ユーザー詳細を再読み込みし、変更結果を確認します。
4. write-audit.jsonl に書き込み監査ログが記録されていることを必要に応じて確認します。

### 5.5 戻し用メモ

更新前に、次の情報を作業メモやチケットに残してください。

- 対象ユーザー sAMAccountName / DN
- 変更日時
- 変更者
- 変更属性
- 変更前の値
- 変更後の値
- 依頼・承認番号

ManageAdTool は戻し操作を自動実行しません。戻す場合も、同じ限定編集手順で変更前の値を入力して実施してください。

---

## 6. コンピュータ検索

1. コンピュータ検索タブを開きます。
2. コンピュータ名、DNS ホスト名、sAMAccountName などで検索します。
3. 必要に応じて OS 名や description 有無などの条件を指定します。
4. 検索結果から対象コンピュータを選択します。

---

## 7. コンピュータ詳細表示

1. コンピュータ検索結果から対象を選択します。
2. コンピュータ名、DNS ホスト名、OS、description、DN、所属グループなどを確認します。
3. 更新前に、対象 DN が `AllowedComputerOuDns` 配下であることを確認します。

---

## 8. コンピュータ description 限定編集

更新対象は **description のみ** です。コンピュータ名、OU、所属グループ、OS 情報などは更新できません。

1. 対象コンピュータを検索・選択します。
2. 新しい description を入力します。
3. 差分を確認します。
4. Domain Admins セッションで再認証します。
5. 確認画面で対象コンピュータ、DN、変更前後を確認します。
6. 更新を実行します。
7. 更新後に詳細を再読み込みし、description が反映されたことを確認します。

空文字による description クリアは対象外です。空にしたい場合でも、本ツールで実施できる操作として扱わないでください。

---

## 9. グループ検索

1. グループ検索タブを開きます。
2. グループ名または DN の一部を入力します。
3. 検索を実行します。
4. 検索結果から対象グループを選択します。

グループ作成、削除、リネームはできません。

---

## 10. グループ詳細表示

1. グループ検索結果から対象グループを選択します。
2. グループ DN、ユーザーメンバー、コンピュータメンバー、ネストグループ、memberOf を確認します。
3. 編集対象にする場合は、`EditableGroupOuDns` 配下で、保護グループに該当しないことを確認します。

---

## 11. ユーザー限定のグループメンバー追加・削除

本ツールで変更できるグループメンバーは **ユーザーの追加・削除のみ** です。

できない操作:

- グループをグループに追加
- コンピュータをグループに追加
- グループ作成・削除・リネーム
- 保護グループの編集

操作手順:

1. 対象グループを検索・選択します。
2. 現在のメンバーを確認します。
3. 追加するユーザーまたは削除するユーザーを指定します。
4. 差分を確認します。
5. Domain Admins セッションで再認証します。
6. 確認画面で対象グループ、対象ユーザー、追加・削除内容を確認します。
7. 更新を実行します。
8. 更新後にグループ詳細を再読み込みし、メンバー状態を確認します。

---

## 12. GPOシミュレーション

GPOシミュレーションは、ユーザー / コンピュータ / 両方を対象に、OU リンクから推定される GPO 一覧を確認する簡易機能です。

確認できる主な項目:

- GPO 名
- GPO ID
- 適用対象
- リンク先 OU
- リンク有効 / 無効
- 強制適用の有無

> **重要**: この機能は簡易シミュレーションであり、GPMC の RSOP（ポリシーの結果セット）や `gpresult /R` の代替ではありません。セキュリティフィルタ、WMI フィルタ、継承ブロック、ループバック処理、サイトリンクなどは考慮しません。

GPO の編集、リンク変更、セキュリティフィルタ変更、WMI フィルタ変更はできません。

---

## 13. オペレーション支援

オペレーション支援タブは、ユーザー所属変更などの作業前に、変更予定を整理するための機能です。

できること:

- 対象ユーザーを検索・選択する
- 属性変更予定を整理する
- グループ追加・削除予定を整理する
- 変更予定サマリーを生成し、クリップボードへコピーする
- 画面上のチェックリストで確認状況を管理する

> **重要**: オペレーション支援タブは **変更予定作成のみ** です。このタブから AD 更新は行いません。実際の更新はユーザー編集またはグループ編集の各機能で、差分確認・再認証・更新後確認を行ってください。

---

## 14. ログ確認

ログ確認タブでは、次の JSON Lines 形式ログを閲覧できます。

| ログ | 内容 |
|---|---|
| `audit.jsonl` | 参照操作、検索、GPOシミュレーション、オペレーション支援の記録 |
| `auth.jsonl` | Domain Admins 認証の成否 |
| `write-audit.jsonl` | AD 更新操作の監査記録 |

主な操作:

1. ログ種別を選択します。
2. 読み込みます。
3. 成否、日付範囲、操作種別、対象名で絞り込みます。
4. 行を選択し、整形済み JSON を確認します。
5. 必要に応じて CSV 出力します。

ログ確認タブは閲覧専用です。ログ編集・削除・改ざん検知・ローテーションは行いません。

---

## 15. CSV出力

検索結果、GPOシミュレーション結果、ログ一覧などは CSV 出力できます。

- 出力先は内部情報を扱える場所を選択してください。
- CSV には AD 属性、DN、操作履歴などの内部情報が含まれる場合があります。
- 文字化けする場合は、Excel の取り込み方法またはエディタの文字コード設定を確認してください。

---

## 16. 注意事項

- 実 AD で更新する前に、必ず検証用 OU で手順を確認してください。
- `AllowedTargetOuDns` / `AllowedComputerOuDns` / `EditableGroupOuDns` が空の場合、対応する更新機能は無効化されます。
- パスワードは appsettings.json やログには保存されません。
- ログや CSV には内部情報が含まれるため、保存先・共有先に注意してください。
- できない操作を、別機能や別属性で代替しないでください。
- 不明点やエラーがある場合は、更新を中断し、管理者に確認してください。
