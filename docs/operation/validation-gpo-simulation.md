# GPOシミュレーション機能 検証手順

v0.8.0 で追加した「GPOシミュレーション」タブの動作を実AD環境で確認する手順です。

## 機能の位置づけと制約

本機能は対象オブジェクトが属する **OU に直接リンクされた GPO を一覧表示する簡易機能**です。
GPMC の「ポリシーの結果セット（RSOP）」や `gpresult /R` の代替にはなりません。

以下の項目は本バージョンでは考慮していません:

- **セキュリティフィルタ**: GPO に設定されたセキュリティフィルタによるフィルタリングは行わない
- **WMIフィルタ**: WMI フィルタの評価は行わない
- **継承ブロック**: OU に設定された「Block Inheritance」は反映しない
- **ループバック処理**: コンピュータ側GPOのユーザー設定ループバックは考慮しない
- **サイトリンク**: AD サイトにリンクされた GPO は対象外

これらを考慮した正確な適用結果が必要な場合は、GPMC の RSOP ウィザードまたは `gpresult /R` を使用してください。

---

## 前提条件

- 起動時ダイアログで `DirectoryReadOnly` または `InMemory` を選択済み
- `LogPath` が書き込み可能なパスに設定済み

---

## 検証項目

### 1. タブ表示

- [ ] 「GPOシミュレーション」タブが画面上に表示される

### 2. 種別選択と入力欄の切り替え

- [ ] 種別「ユーザー」選択時: sAMAccountName 欄のみ表示される
- [ ] 種別「コンピュータ」選択時: コンピュータ名欄のみ表示される
- [ ] 種別「ユーザー + コンピュータ」選択時: 両方の入力欄が表示される

### 3. ユーザー種別でのシミュレーション実行

- [ ] sAMAccountName に実在するユーザーを入力し「シミュレーション実行」を押すと、GPO一覧が表示される
- [ ] GPO名 / GPO ID / 適用対象 / リンク先OU / 有効 / 強制適用 / 備考 が表示される
- [ ] ユーザーが属するOUとその上位OU（ドメインルートまで）にリンクされたGPOが一覧に含まれている（ADUCやGPMCのリンク情報と照合）
- [ ] 存在しない sAMAccountName を入力した場合、エラーメッセージが表示される

### 4. コンピュータ種別でのシミュレーション実行

- [ ] コンピュータ名に実在するコンピュータを入力し「シミュレーション実行」を押すと、GPO一覧が表示される
- [ ] コンピュータが属するOUとその上位OU（ドメインルートまで）にリンクされたGPOが確認できる
- [ ] 存在しないコンピュータ名を入力した場合、エラーメッセージが表示される

### 5. ユーザー + コンピュータ種別でのシミュレーション実行

- [ ] 両方の入力欄に有効な値を入力してシミュレーション実行すると、両方のOU経路のGPOが一覧表示される
- [ ] どちらか一方が空の場合はバリデーションエラーが表示される

### 6. GPO情報の確認

- [ ] リンク先OU の値が実際の AD 構造と一致している（ADUCやGPMCで確認）
- [ ] 「有効」列: リンク無効（link disabled）のGPOが「いいえ」と表示される
- [ ] 「強制適用」列: Enforced 設定のGPOが「はい」と表示される
- [ ] 「備考」列: リンク無効・GPO全体無効の場合に注記が表示される

> **注**: セキュリティフィルタ・WMIフィルタ・継承ブロック・サイトリンクは反映されないため、
> 実際に適用されないGPOが一覧に含まれる場合があります。

### 7. 結果コピー

- [ ] 「結果をコピー」ボタンを押すと、テキスト形式でクリップボードにコピーされる
- [ ] テキストエディタに貼り付けて GPO名・リンク先OU・有効・強制適用が含まれることを確認する
- [ ] 結果が0件の状態でコピーボタンを押した場合、エラーメッセージが表示される

### 8. CSV出力

- [ ] 「CSV出力」ボタンを押すと、保存ダイアログが表示される
- [ ] CSVファイルを保存し、Excel などで開いて内容を確認する（ヘッダー7列 + データ行）
- [ ] BOM付きUTF-8で出力されており、日本語が文字化けしないことを確認する

### 9. AD更新が実行されないこと（最重要）

- [ ] GPOシミュレーションタブのいかなる操作も、ADへの書き込みを行わない
- [ ] シミュレーション実行後に `write-audit.jsonl` を確認し、このタブの操作に起因する更新ログが記録されていないことを確認する

### 10. 参照ログへの記録

- [ ] シミュレーション実行後に `audit.jsonl` を確認し、`"action": "GpoSimulation"` のエントリが記録されている
- [ ] ログに `executor` / `simulationType` / `targetUser` / `targetComputer` / `resultCount` が含まれる
- [ ] CSV出力時に `"action": "GpoSimulationCsvExport"` が記録される

### 11. InMemory モードでの動作確認

- [ ] 起動時ダイアログで `InMemory` を選択 でもシミュレーションが実行でき、デモGPOデータが表示される
- [ ] InMemory で存在しないユーザー名を入力した場合、エラーメッセージが表示される
