# グループメンバー編集 検証手順書 (v0.6.0)

## 概要

本手順書は ManageAdTool v0.6.0 のグループメンバー追加・削除機能を実 AD 環境で検証するための手順を記述します。

**更新対象**: グループへのユーザー追加・削除（`member` 属性）のみ  
**禁止操作**: グループ作成・削除・リネーム / グループをグループに追加 / コンピュータをグループに追加 / GPO編集 / OU移動 / 一括更新  
**前提条件**: `DirectoryReadOnly` モード + Domain Admins セッション + `EditableGroupOuDns` 設定

---

## 事前準備

### appsettings.json の設定

`config-samples/appsettings.GroupMembershipEdit.sample.json` を参考に設定する。

```json
{
  "AppPolicy": {
    "ServiceMode": "DirectoryReadOnly",
    "EditorAuthMode": "DomainAdmins",
    "AdminGroupDn": "CN=Domain Admins,CN=Users,DC=example,DC=local",
    "EditableGroupOuDns": [
      "OU=Groups,DC=example,DC=local"
    ],
    "ProtectedGroupNames": ["Domain Admins", "Enterprise Admins", "Schema Admins", "Administrators"],
    "ProtectedGroupDns": [
      "CN=Domain Admins,CN=Users,DC=example,DC=local"
    ]
  }
}
```

> **EditableGroupOuDns の設定について**: `EditableGroupOuDns` が空の場合はグループメンバーの更新は実行できません（セーフガード）。特定のOUを明示的に指定することで、意図しないグループへの変更を防止します。

> **ProtectedGroupNames / ProtectedGroupDns の設定について**: 特権グループ（Domain Admins 等）は必ずここに登録してください。登録されたグループはメンバー追加・削除不可になります。

### テスト用グループ・ユーザー

AD 上に以下のテスト環境を用意する:

- `GG_TestGroup`（検証用グループ、`EditableGroupOuDns` 内に配置）
- テスト用ユーザー `test.user01`、`test.user02`

## 検証項目

### 1. グループ検索

| 項目 | 手順 | 期待結果 |
|------|------|---------|
| 基本検索 | 「グループ参照・メンバー編集」タブでグループ名の一部（2文字以上）を入力して「検索」 | 該当グループが一覧に表示される |
| 1文字検索禁止 | 1文字を入力して「検索」 | 「2文字以上入力してください」メッセージ |
| 上限到達 | 広範な検索語で MaxSearchResults 件数に達する場合 | 上限件数と超過メッセージを表示 |

### 2. グループ詳細表示

| 項目 | 手順 | 期待結果 |
|------|------|---------|
| 詳細表示 | 検索結果からグループを選択 | 左ペインに Name / DN / Description が表示される |
| ユーザーメンバー表示 | 同上 | ユーザーメンバー DataGrid に SamAccountName / DisplayName / Mail / Department / Enabled が表示される |
| コンピュータ・ネストグループ表示 | 同上 | 下部テキストにコンピュータメンバー数・ネストグループ数・所属グループ（memberOf）が表示される |

### 3. 編集ブロック確認

| 項目 | 手順 | 期待結果 |
|------|------|---------|
| 未選択時 | グループ未選択 | 追加・削除ボタン入力不可、「グループ未選択」表示 |
| 未ログイン時 | ログインせずにグループ選択 | 入力不可、理由が表示される |
| EditableGroupOuDns 未設定 | 設定を空にしてグループ選択 | 入力不可、「EditableGroupOuDns 未設定のため更新不可」と表示される |
| 許可OU外 | 設定外 OU のグループを選択 | 入力不可、「対象グループが許可OU外」と表示される |
| 保護グループ名 | ProtectedGroupNames に登録されたグループを選択 | 入力不可、「保護グループ名のため編集不可」と表示される |
| 保護グループ DN | ProtectedGroupDns に登録されたグループを選択 | 入力不可、「保護グループのため編集不可」と表示される |

### 4. ユーザー追加フロー

| # | 手順 | 期待結果 |
|---|------|---------|
| 1 | Domain Admins アカウントでログイン | セッション開始、追加予定欄が有効化される |
| 2 | 追加するユーザーのSAMを入力して「検索して追加予定へ」 | ADからユーザーを検索し、追加予定リストに表示される |
| 3 | 「差分確認」をクリック | OutputBox に追加・削除予定の一覧が表示される。「限定更新実行」ボタンが有効化される |
| 4 | 「限定更新実行」をクリック | 再認証ダイアログが開く |
| 5 | 正しい Domain Admins 資格情報を入力 | ConfirmGroupMemberUpdateDialog が開く。グループ名・DN・実行端末・編集者・追加ユーザー（緑）が表示される |
| 6 | 「この内容でADを更新する」をクリック | AD更新が実行され、ユーザーがグループメンバーになる |
| 7 | OutputBox を確認 | 更新成功メッセージ・AD再取得後のメンバー数が表示される |
| 8 | write-audit.jsonl を確認 | `operationName: "UpdateGroupMembers"`, `targetType: "Group"`, `targetName`, `changes[].ldapAttribute: "member"` が記録されていること |

### 5. ユーザー削除フロー

| # | 手順 | 期待結果 |
|---|------|---------|
| 1 | グループを選択し、ユーザーメンバー DataGrid で削除するユーザーを選択 | 行がハイライトされる |
| 2 | 「選択を削除予定へ」ボタンをクリック | 削除予定リストにユーザーが追加される |
| 3 | 「差分確認」をクリック | 削除予定の一覧が OutputBox に表示される |
| 4 | 「限定更新実行」→ 再認証 → 確認ダイアログ | 削除するユーザー（赤）が表示される |
| 5 | 「この内容でADを更新する」をクリック | AD更新が実行され、ユーザーがグループから削除される |

### 6. 禁止操作の UI 不在確認

以下の操作が UI に一切存在しないことを目視で確認する。

| 禁止操作 | 確認内容 |
|---------|---------|
| グループ作成 | グループ作成ボタン・ダイアログがないこと |
| グループ削除 | グループ削除ボタンがないこと |
| グループリネーム | グループ名変更のコントロールがないこと |
| グループをグループに追加 | グループ種別のメンバー追加UIがないこと |
| コンピュータをグループに追加 | コンピュータ追加UIがないこと |
| GPO編集 | GPO関連のコントロールが一切ないこと |
| OU移動 | OU移動ボタン・フィールドがないこと |
| 一括更新 | 複数グループを一括更新するUIがないこと |

### 7. SAM不在ユーザーの追加試行禁止

| 項目 | 手順 | 期待結果 |
|------|------|---------|
| 存在しない SAM を追加 | 存在しない SamAccountName を入力して「検索して追加予定へ」 | 「ADに見つかりません」メッセージ、追加予定に追加されない |

### 8. 既存メンバー重複追加禁止

| 項目 | 手順 | 期待結果 |
|------|------|---------|
| 既存メンバーを追加予定へ | グループに既に存在するユーザーのSAMを入力して追加 | 「既にグループメンバーです」メッセージ |

### 9. 矛盾する操作の禁止

| 項目 | 手順 | 期待結果 |
|------|------|---------|
| 追加予定と削除予定に同一ユーザー | 追加予定のユーザーを削除予定へ追加（またはその逆） | 「矛盾する操作は追加できません」メッセージ |

### 10. 整合性チェック

| 項目 | 手順 | 期待結果 |
|------|------|---------|
| 差分確認後にAD側でメンバー変更 | 差分確認後、別の手段でグループメンバーを変更してから「限定更新実行」 | 「既にメンバーです」または「メンバーではありません」メッセージ、更新中止 |

### 11. セッション期限切れ

| 項目 | 手順 | 期待結果 |
|------|------|---------|
| 期限切れ後の更新試行 | EditSessionMinutes 経過後に「限定更新実行」 | 「セッション期限切れ」メッセージ、更新ブロック |

### 12. 再認証失敗

| 項目 | 手順 | 期待結果 |
|------|------|---------|
| 誤パスワードで再認証 | 再認証ダイアログで誤ったパスワードを入力 | 「再認証に失敗しました」メッセージ |
| 非 Domain Admins で再認証 | Domain Admins 非メンバーで再認証 | 「Domain Admins のメンバーではありません」メッセージ |
| セッションユーザーと不一致 | セッション開始者と異なるアカウントで再認証 | 「再認証ユーザーと編集セッションユーザーが一致しません」メッセージ |

### 13. キャンセル動作

| 項目 | 手順 | 期待結果 |
|------|------|---------|
| 再認証ダイアログキャンセル | 再認証ダイアログで「キャンセル」 | 更新中止、AD更新は実行されない |
| 確認ダイアログキャンセル | ConfirmGroupMemberUpdateDialog で「キャンセル」 | 更新中止 |

## 確認後の処理

- 検証に使用したグループのメンバーを元の状態に戻す
- write-audit.jsonl の `operationName: "UpdateGroupMembers"` エントリを参照して変更内容を確認する
- 検証結果を `docs/operation/test-record-v0.6.0.md` に記録する（別途作成）
